# Test Execution Reports

## Overview

Test execution reports summarize testing activities, results, and findings.

## Report Components

### Executive Summary
- Testing scope
- Overall status
- Key findings
- Risks and recommendations

### Test Results
- Tests executed
- Pass/fail counts
- Coverage achieved
- Defects found

### Defects
- New bugs found
- Bug severity distribution
- Blocking issues

### Recommendations
- Go/no-go decision
- Risk areas
- Additional testing needed

## Report Format

See [Test Report Template](../reference-templates/test-report-template.md).

## When to Report

- Daily status during active testing
- Weekly summaries
- Release verification complete
- After major test milestones

## Best Practices

- Be concise and factual
- Highlight risks clearly
- Provide data and trends
- Include recommendations
- Distribute to stakeholders
