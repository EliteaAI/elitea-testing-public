## Instructions

!!! tip How to Use This Template
    1. Copy this template for each release cycle 
    2. Fill in the Release Information section above 
    3. Test each scenario systematically 
    4. Mark Status: **P** (Pass), **F** (Fail), **NA** (Not Applicable), **B** (Blocked) 
    5. Document any failures or issues in the Notes/Bug ID column 
    6. Link to bug reports using format: `#issue-number` or full URL
    7. Complete all applicable tests before release sign-off

!!! warning "Critical Testing Areas"
    Areas marked with 🔴 are **critical path** and must pass for release approval.
