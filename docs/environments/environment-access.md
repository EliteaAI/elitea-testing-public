# Environment Access

## Overview

This guide explains how to request and obtain access to Elitea testing environments.

## Requesting Access

### New Team Members
1. Contact QA team lead
2. Provide your GitHub username and email
3. Specify which environments needed (DEV, STAGE)
4. Receive access credentials

### Temporary Access
For contractors or short-term access:
1. Submit access request
2. Specify duration and purpose
3. Receive temporary credentials

## Access Credentials

Credentials are provided securely through:
- Team password manager
- Secure credential sharing tool
- Direct communication from QA lead

**Never:**
- Share credentials publicly
- Commit credentials to repositories
- Store credentials in unsecured locations

## Account Types

### QA Test Accounts
- Dedicated accounts for testing
- Multiple roles available
- Specific to each environment

### Personal Accounts
- For team members
- Administrative access
- Use for configuration, not testing

## Troubleshooting Access Issues

- Verify VPN connection if required
- Check account status
- Contact QA team lead
- Review access permissions
