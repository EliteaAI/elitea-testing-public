# Test Accounts

## Overview

Test accounts are dedicated user accounts used for testing across different environments.

## Account Categories

### Standard User Accounts
- Basic user role
- Standard permissions
- Used for typical user workflow testing

### Admin Accounts
- Administrative role
- Full permissions
- Used for admin feature testing

### Role-Specific Accounts
- Specific roles (viewer, contributor, etc.)
- Limited permissions
- Used for permission testing

## Accessing Test Account Credentials

**Security Policy:**
Test account credentials are stored securely and NOT included in this public documentation.

**To access credentials:**
1. Contact the QA team lead
2. Access the team password manager
3. Reference the secure credentials document

## Using Test Accounts

### Best Practices
- Use appropriate account for test scenario
- Don't modify account settings unnecessarily
- Report any account issues
- Keep credentials secure

### Naming Convention
Test accounts follow the pattern:
- `qa_user_<number>` for standard users
- `qa_admin_<number>` for admin users
- `qa_<role>_<number>` for role-specific users

## Test Account Management

### Creating New Test Accounts
Contact QA lead to request new test accounts with specific roles or configurations.

### Resetting Passwords
If a test account password needs reset, contact QA lead.
