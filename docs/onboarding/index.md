# Onboarding

Welcome to the Elitea QA team! This section provides resources to help you get started.

## Getting Started

### First Week

#### Day 1
- [ ] Get access to environments (see [Environment Access](../environments/environment-access.md))
- [ ] Get added to team communication channels
- [ ] Review this handbook
- [ ] Meet the team

#### Day 2-3
- [ ] Read core documentation:
  - [Bug Reporting Guide](../bug-management/bug-reporting-guide.md)
  - [Bug Priority and Severity](../bug-management/bug-priority-severity.md)
  - [Labels and Taxonomy](../bug-management/labels-taxonomy.md)
- [ ] Get access to ELITEA Board
- [ ] Review recent bugs and test cases

#### Day 4-5
- [ ] Shadow experienced QA engineer
- [ ] Execute sample test cases
- [ ] Report a test bug (with guidance)
- [ ] Learn test automation setup (if applicable)

### First Month

- [ ] Execute tests independently
- [ ] Report bugs following standards
- [ ] Participate in bug triage
- [ ] Contribute to test planning
- [ ] Learn automation basics
