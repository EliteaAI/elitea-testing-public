# Manual Testing

## Overview

Manual testing is human-driven exploration and verification of software functionality.

## When to Use Manual Testing

- New features without automation
- Exploratory testing
- Usability and UX testing
- Complex scenarios
- One-time verification

## Manual Testing Process

### 1. Preparation
- Review test cases
- Set up test environment
- Prepare test data
- Understand acceptance criteria

### 2. Execution
- Follow test steps systematically
- Document actual results
- Take screenshots for evidence
- Note any deviations or observations

### 3. Documentation
- Record pass/fail for each test
- File bugs for failures
- Note any test case updates needed
- Provide summary of results

## Best Practices

- Be methodical and thorough
- Document as you test
- Use consistent test data
- Take clear screenshots
- Report issues immediately
- Don't rush through tests

## Tools

- Screen capture: Screenshots, recordings
- Note taking: Test results spreadsheet
- Bug tracking: GitHub Issues
