# Test Results Documentation

## Overview

Documenting test results provides traceability, enables analysis, and communicates quality status.

## What to Document

### Test Execution Records
- Test case ID and title
- Execution date and time
- Tester name
- Environment (DEV, STAGE)
- Pass/fail status
- Comments and observations

### Evidence
- Screenshots for failures
- Logs and error messages
- Actual vs expected results
- Steps to reproduce issues

### Summary Metrics
- Total tests executed
- Pass/fail counts
- Defect counts
- Coverage percentage

## Documentation Formats

### Test Execution Sheet
Spreadsheet or table with:
- Test case details
- Results
- Comments
- Links to bugs

### Test Report
Summary document including:
- Testing scope
- Environment
- Execution summary
- Defects found
- Risks and recommendations

## Where to Store Results

- Test management system (if available)
- Shared documentation
- Linked to test cases
- Referenced in bug reports

## Best Practices

- Document as you test
- Be specific and factual
- Include timestamps
- Link to evidence
- Summarize key findings
- Communicate blockers immediately
